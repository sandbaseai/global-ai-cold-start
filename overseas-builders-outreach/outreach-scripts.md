# Outreach Scripts

These are starting points. Every message should be personalized before sending.

## X / LinkedIn Short Message

```text
Hey [Name], saw your work on [specific project/post].

We are building SandBase, agent infrastructure for developers building production AI agents.

The part I am especially curious about is [specific angle: sandboxed tools / runtime / browser agents / retries / observability].

How are you handling that today?
```

## Open-Source Maintainer

```text
Hi [Name],

I found [repo] while looking at projects around AI agent runtime and sandboxing.

The interesting part to me is [specific detail].

We are building SandBase around sandboxed runtime, safe tool execution, model routing, and APIs for production AI agents.

If useful, I would be happy to test the project, write a short integration note, or compare notes on what usually breaks when agents move from demos to production.
```

## Founder / Indie Hacker

```text
Hey [Name],

I saw you are building [product]. The agent workflow looks close to a problem we are studying: how to run tool-using agents safely once they leave the demo environment.

We are building SandBase as runtime infrastructure for production AI agents.

No hard pitch. I am trying to learn how builders are handling sandboxing, tool execution, retries, and cost limits today.

Would be curious to hear what has been painful so far.
```

## Follow-Up

```text
Quick follow-up on this.

Even if SandBase is not relevant right now, I would still be curious:

What is the hardest part of moving an agent from demo to production in your workflow?
```

## When They Reply With Interest

```text
That makes sense.

The pattern we keep seeing is that the hard part is not only the model call. It is runtime: tool isolation, logs, retries, budget limits, and cleaning up execution environments.

If helpful, I can share a short checklist we are using to evaluate production-agent readiness.
```
