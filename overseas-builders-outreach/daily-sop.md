# Daily SOP — Overseas Builders Outreach

This SOP is the daily operating loop for reaching overseas AI builders without becoming spam.

Time budget: 60-90 minutes per day.

## Daily Outcome

Every day should produce:

- 10 new relevant leads found
- 3-5 leads selected for outreach
- 1-3 personalized messages sent or prepared
- all replies checked
- one product or content learning recorded
- next actions updated

## Inputs

Use these sources:

- X search
- LinkedIn search
- GitHub repositories
- Peerlist posts
- Dev.to articles
- Hacker News launches
- Reddit technical threads
- hackathon project galleries
- newsletters and curated AI agent lists

## Step 1 — Search

Search for builders around these keywords:

```text
AI agent runtime
agent sandbox
tool calling
browser agent
coding agent
MCP tools
agent observability
production AI agents
multi-agent workflow
AI agent infrastructure
```

## Step 2 — Qualify

For each candidate, ask:

- Are they actively building or discussing AI agents?
- Do they use tools, browser, shell, files, APIs, or code execution?
- Are they likely to care about reliability, sandboxing, cost limits, or observability?
- Is there a natural SandBase angle?
- Is there a reachable public channel?

## Step 3 — Score

Use the CRM table:

| Score | Meaning |
|-------|---------|
| 5 | Excellent fit |
| 4 | Strong fit |
| 3 | Possible fit |
| 2 | Weak fit |
| 1 | Not worth outreach |

Fields:

- Fit
- Reachability
- Urgency
- SandBase angle

Only prepare outreach if:

```text
Fit >= 4
SandBase angle >= 4
Reachability >= 3
```

## Step 4 — Personalize

Every first message must include:

1. Specific project, post, or repo reference.
2. One technical observation.
3. One relevant question.
4. No hard ask in the first sentence.

Bad:

```text
We built SandBase. Can you try it?
```

Better:

```text
I saw your browser-agent project. Curious how you are isolating browser/file access once tasks run longer than a demo. Is that handled in the app layer today?
```

## Step 5 — Send / Prepare

Send only if the message is specific.

If not enough context exists, prepare the message but do not send yet.

## Step 6 — Follow Up

Follow-up rules:

- First follow-up after 3-5 days.
- One useful follow-up only.
- Do not send more than 2 messages without a reply.
- Follow-up should ask a useful technical question, not repeat the pitch.

## Step 7 — Record Learning

Every day, write one learning:

- repeated pain point
- unclear positioning
- new content idea
- product feature signal
- objection
- channel quality signal

## Daily Checklist

- [ ] Checked replies.
- [ ] Found 10 leads.
- [ ] Scored leads.
- [ ] Selected 3-5 high-fit leads.
- [ ] Prepared personalized outreach.
- [ ] Sent approved messages.
- [ ] Recorded follow-ups.
- [ ] Captured one product/content learning.

```md
- Messages sent:

- Qualified conversations:
- Follow-ups due:
- Product/content learning:
- Tomorrow priority:
```
