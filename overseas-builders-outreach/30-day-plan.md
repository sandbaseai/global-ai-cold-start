# 30-Day Overseas Builders Outreach Plan

Goal: turn public trust into real conversations with overseas AI builders, early users, partners, and commercial opportunities.

This is not a spam campaign.

The goal is to find builders already working on AI agents, understand their workflow, and offer useful infrastructure help where SandBase is relevant.

## North Star

```text
Talk to builders before trying to sell to builders.
```

## Target Builders

Prioritize people who are already close to the problem:

- AI agent startup founders
- indie hackers building agent products
- open-source maintainers in agent runtime, sandboxing, MCP, browser automation, tool calling, workflow engines, or observability
- developers posting about AI agents on X, LinkedIn, Peerlist, Reddit, Hacker News, and GitHub
- teams building coding agents, research agents, browser agents, data agents, or workflow agents
- developer community organizers with AI builder audiences

## Offer

Lead with help, not a pitch.

Useful offers:

- review an agent architecture
- share a sandbox/runtime checklist
- provide early SandBase credits
- help test tool execution workflows
- feature a builder project in the SandBase ecosystem
- invite builders to Discord for technical feedback
- write a short technical teardown or case study with permission

## 30-Day Arc

| Week | Theme | Goal |
|------|-------|------|
| Week 1 | List building and message testing | Build the first 100-target list and test 3 outreach angles |
| Week 2 | First conversations | Start 10-15 useful conversations and identify repeated pain points |
| Week 3 | Partnership and conversion paths | Turn interested builders into demos, Discord members, credits users, or technical collaborations |
| Week 4 | Repeatable outreach system | Create templates, CRM workflow, follow-up loops, and public lessons |

## Daily Operating Loop

Use the [Daily SOP](daily-sop.md) as the repeatable operating checklist.

Spend 45-90 minutes per day:

1. Find 10 relevant builders or projects.
2. Score each lead.
3. Pick 3-5 high-fit people.
4. Send or prepare personalized outreach.
5. Reply to existing conversations.
6. Record the status and next action.
7. Extract one learning into content or product feedback.

## Lead Scoring

| Field | Score |
|-------|-------|
| Fit | 1-5 |
| Reachability | 1-5 |
| Urgency | 1-5 |
| SandBase angle | 1-5 |
| Notes | Free text |

High-priority lead:

```text
Fit >= 4
Reachability >= 3
Urgency >= 3
SandBase angle >= 4
```

## Outreach Angles

### Runtime Angle

For builders shipping long-running agents:

```text
I saw you are building around AI agents. Curious how you are handling runtime concerns like sandboxed tool execution, timeouts, retries, and logs once agents move beyond demos.
```

### Sandbox Angle

For builders running code, browser, shell, or file tools:

```text
Your project looks close to the problem we care about: safe tool execution for AI agents. Are you already isolating browser/shell/file access, or is that still handled inside the app layer?
```

### Builder Support Angle

For early-stage founders and indie hackers:

```text
We are supporting a small number of AI agent builders with runtime/sandbox feedback. If helpful, I can take a look at your agent architecture and share what usually breaks when it moves toward production.
```

### Community Angle

For people discussing agent infrastructure publicly:

```text
This is exactly the kind of production-agent problem we are tracking: not model quality, but runtime reliability and tool boundaries. Would love to compare notes if you are working on this.
```

## Do Not Do

- Do not mass-DM generic pitches.
- Do not pretend to be a user.
- Do not hide that SandBase is the company behind the outreach.
- Do not overpromise product readiness.
- Do not ask for a demo before understanding the builder's problem.
- Do not publish private conversation details without permission.
