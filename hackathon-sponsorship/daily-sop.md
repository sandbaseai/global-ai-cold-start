# Daily SOP — Hackathon Sponsorship

This SOP is the daily operating loop for finding AI hackathons and builder events where SandBase can provide real technical value.

Time budget: 45-75 minutes per day.

## Daily Outcome

Every day should produce:

- 3-5 candidate events found
- 1-2 events selected for outreach
- organizer contact path identified
- one personalized message prepared or sent
- follow-up status updated

## Inputs

Use these sources:

- Devpost
- Lablab.ai
- DoraHacks
- HackQuest
- Eventbrite
- Meetup
- university AI clubs
- developer community newsletters
- X / LinkedIn event posts
- AI agent communities

## Step 1 — Search

Search around:

```text
AI agent hackathon
AI agents challenge
LLM hackathon
developer hackathon AI
AI builder event
AI infra hackathon
agentic AI hackathon
open source AI hackathon
```

## Step 2 — Qualify

For each event, ask:

- Are builders writing real code?
- Is there an AI agent / devtool / automation angle?
- Does the timeline fit the next 1-4 months?
- Is there a sponsor or community partner path?
- Can SandBase provide useful credits, templates, or office hours?
- Would participants benefit from sandboxed runtime or tool execution?

## Step 3 — Score

Fields:

- AI agent relevance
- Builder quality
- Sponsor accessibility
- Timeline fit
- SandBase fit

Only prioritize if:

```text
AI agent relevance >= 4
Builder quality >= 4
SandBase fit >= 4
```

## Step 4 — Prepare Offer

Default offer:

- SandBase credits
- starter templates
- office hours
- prize track
- post-event showcase

Prize track:

```text
Best Production-Ready Agent
```

## Step 5 — Contact Organizer

Every message must include:

1. Specific event reference.
2. Why SandBase fits the event.
3. Clear support package.
4. Lightweight ask.

Example:

```text
I saw [event] is focused on AI agents. We are building SandBase, agent infrastructure for production AI agents.

We can support teams with sandboxed runtime credits, starter templates, and office hours for tool-using agents.

Would it be useful if I sent a lightweight support package for your hackathon teams?
```

## Step 6 — Follow Up

Follow-up rules:

- First follow-up after 5-7 days.
- Include a concrete support idea.
- Do not push paid sponsorship if the event is not a fit.

## Daily Checklist

- [ ] Found 3-5 events.
- [ ] Scored events.
- [ ] Picked 1-2 high-fit events.
- [ ] Found organizer contact.
- [ ] Prepared organizer message.
- [ ] Sent approved messages.
- [ ] Updated follow-up status.
- [ ] Recorded one event/channel learning.

## Daily Output Format

```md
## Daily Summary

- Events found:
- High-fit events:
- Messages prepared:
- Messages sent:
- Replies:
- Follow-ups due:
- Sponsorship learning:
- Tomorrow priority:
```
