# Public Promotion Plan

This plan covers the first public push for two SandBase open-source assets:

1. [Awesome Agent Runtime](https://github.com/sandbaseai/awesome-agent-runtime)
2. [Global AI Cold Start](https://github.com/sandbaseai/global-ai-cold-start)

Promote the resource value first. SandBase.ai should appear as the builder and maintainer behind the assets, not as a hard product pitch.

## Order Of Promotion

### 1. Awesome Agent Runtime

Primary audience:

- AI agent builders
- infrastructure engineers
- maintainers of agent frameworks, sandboxes, MCP servers, observability tools, and deployment platforms
- developer-tool communities

Why it should go first:

- It is useful even to people who do not know SandBase.
- It creates a reason for maintainers to respond.
- It can earn stars, issues, corrections, and backlinks naturally.
- It positions SandBase inside the production AI agent infrastructure conversation.

Primary message:

```text
We mapped 500 projects across the production AI agent infrastructure stack.
```

### 2. Global AI Cold Start

Primary audience:

- AI founders
- indie hackers
- technical founders doing overseas growth
- builders using AI agents or Codex for operations
- startup operators who want a repeatable public-trust playbook

Why it should go second:

- It is a founder/operator story.
- It explains how the public trust layer was built.
- It works best after the open-source asset has already created proof.

Primary message:

```text
We documented the first 20 days of turning an AI infrastructure startup from almost invisible into a public, searchable trust surface.
```

## Community Rules

- Do not post the same link everywhere with the same copy.
- Lead with one useful observation before linking.
- Prefer maintainer corrections over generic promotion.
- Do not imply listed projects endorse SandBase.
- Track every useful reply, correction, and question as product/content signal.

## Success Signals

Early signals:

- stars
- forks
- issue suggestions
- maintainer corrections
- newsletter/resource-list mentions
- category discussions
- backlinks from project docs, READMEs, blogs, or community posts

Avoid treating impressions alone as the main metric.
