## 目标

让 GitHub resource repo 本身足够有用，而不是只作为一个外链。

## 计划工作

- 从开发者视角检查当前 resource repo。
- 优化 README 结构、分类和定位。
- 如果有必要，增加贡献说明。
- 自然链接 SandBase，不让 repo 显得像广告。
- 记录这个 repo 未来如何变得更有用。
